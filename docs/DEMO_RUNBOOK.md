## Act 2: "The Agentic Workflow" (AI-Powered Pipeline Creation)

**Story**: *"Now here's where it gets interesting. Instead of a data engineer manually writing Bronze, Silver, and Gold transformations, our AI agent discovers the data and builds the entire pipeline autonomously."*

### Step 3: Run the Agentic Workflow

```sql
-- This is the main event! The agent will:
-- 1. TRIGGER: Discover new tables in the landing zone
-- 2. PLAN: Use Cortex LLM to analyze schemas and plan transformations
-- 3. EXECUTE: Generate and execute Dynamic Table DDL (Bronze → Silver → Gold)
-- 4. VALIDATE: Verify data quality and row counts
-- 5. REFLECT: Log learnings for future improvements
-- 6. PROPAGATE TO GOLD: Build aggregation tables and Semantic Views

CALL DBAONTAP_ANALYTICS.AGENTS.RUN_AGENTIC_WORKFLOW('manual');
```

> **Note**: This takes 2-5 minutes. While waiting, show the Streamlit app to monitor progress.

### Step 4: Monitor via Streamlit

Open the **DEMO_MANAGER** Streamlit app in Snowsight:
- Navigate to: `DBAONTAP_ANALYTICS.METADATA.DEMO_MANAGER`
- Show the **Workflow** tab to see real-time agent phases
- Show the **Pipeline Health** tab for row counts across layers

### Step 5: Verify the pipeline was built

```sql
-- Check Dynamic Tables created across all layers
SHOW DYNAMIC TABLES IN DATABASE DBAONTAP_ANALYTICS;

-- Pipeline health check
SELECT 'LANDING' as stage, 'customers' as entity, COUNT(*) as rows FROM "public"."customers"
UNION ALL SELECT 'BRONZE', 'customers_variant', COUNT(*) FROM BRONZE.CUSTOMERS_VARIANT
UNION ALL SELECT 'SILVER', 'customers', COUNT(*) FROM SILVER.CUSTOMERS
UNION ALL SELECT 'GOLD', 'customer_360', COUNT(*) FROM GOLD.CUSTOMER_360;

-- Check what the agent decided (LLM reasoning)
SELECT execution_id, source_table, transformation_strategy, llm_reasoning
FROM METADATA.PLANNER_DECISIONS
ORDER BY created_at DESC;

-- Check validation results
SELECT source_table, target_table, validation_type, passed, details
FROM METADATA.VALIDATION_RESULTS
ORDER BY created_at DESC;
```

**Talking Point**: *"The agent used Cortex LLMs to analyze the raw data, determine the right transformation strategy, generate Dynamic Table DDL, validate the results, and even reflected on what it learned. Zero manual SQL authoring."*

## Act 5: "Schema Evolution" (Optional Bonus)

**Story**: *"What happens when the application team adds a new column? Traditional pipelines break. Ours adapts."*

### Step 8: Add a column to Postgres SOURCE

```bash
psql "service=dbaontap_source"
```

```sql
-- Add a loyalty tier column
ALTER TABLE customers ADD COLUMN loyalty_tier VARCHAR(20) DEFAULT 'bronze';

-- Update some customers
UPDATE customers SET loyalty_tier = 'gold' WHERE segment = 'enterprise';
UPDATE customers SET loyalty_tier = 'silver' WHERE segment = 'mid-market';
```

### Step 9: Run the workflow again to detect schema change

```sql
-- The agent detects the new column and evolves the pipeline
CALL DBAONTAP_ANALYTICS.AGENTS.RUN_AGENTIC_WORKFLOW('manual');
```

### Step 10: Verify schema propagation

```sql

DESCRIBE DYNAMIC TABLE SILVER.CUSTOMERS;

-- Check Gold propagated the change

-- Semantic Views regenerated with new dimension
SHOW SEMANTIC VIEWS IN SCHEMA GOLD;
```

**Talking Point**: *"The agent detected the schema change, propagated it through Bronze → Silver → Gold, regenerated the Semantic Views, and updated the Knowledge Graph — all autonomously."*

## Troubleshooting

| Issue | Fix |
|-------|-----|
| CDC data not appearing in Snowflake | Check Openflow services: `SHOW SERVICES IN ACCOUNT;` |
| Dynamic Tables not refreshing | Check state: `SHOW DYNAMIC TABLES IN DATABASE DBAONTAP_ANALYTICS;` — look for SUSPENDED |
| Streams stale | Recreate: `CREATE OR REPLACE STREAM <schema>.<name> ON TABLE "public"."<table>";` |
| Workflow procedure fails | Check logs: `SELECT * FROM METADATA.WORKFLOW_EXECUTIONS ORDER BY started_at DESC LIMIT 5;` |
| Semantic Views missing | Run: `CALL AGENTS.RUN_SEMANTIC_VIEW_PIPELINE();` |
| Postgres instances suspended | Resume: `ALTER POSTGRES INSTANCE <name> RESUME;` |
